ToolRouter

A hierarchical tool router for LLM agents with too many tools. Narrow the search space step-by-step and let the agent safely return no_tool_available.

View on GitHub Quickstart

The Problem

Flat tool lists get noisy fast.

Once agents see dozens or hundreds of tools at once, selection gets expensive, brittle, and harder to debug.

Less context bloat

Each routing step includes only the current path and available child options.

Safer refusal

no_tool_available is a normal result, not an afterthought.

Auditable traces

Every decision records path, options, votes, confidence, and reason.

Routing Shape

Ask for one constrained choice at a time.

ToolRouter traverses whatever tree you define, one level at a time, until it reaches a final tool or safely returns no tool. Multi-sample voting and confidence thresholds help avoid low-confidence calls. 

const router = createToolRouter({
  confidenceThreshold: 0.82,
  samples: 3,
  allowNoTool: true,
  llm,
  tree,
  tools
});

const result = await router.route({
  request: "Summarize recent papers..."
});
